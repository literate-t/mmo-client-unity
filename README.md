# 2D MMORPG Game Client

## 구현 기능

### 타일 맵의 레이어
- 게임 오브젝트가 배치된 순서대로 렌더링되기 때문에 기본 맵, 맵의 환경, 맵의 충돌 게임 오브젝트를 각각 나누어서 게임 오브젝트를 생성

### 툴 에디터
- 맵의 충돌 게임 오브젝트를 통해, 충돌 정보를 0과 1로 표현한 텍스트 문서를 생성할 수 있다
    - 이 텍스트 문서는 서버에서 액터들의 충돌 체크에 사용한다
- 실행 파일을 생성하고 실행할 수 있다

### 조작
- 결정형 FSM을 사용해 switch 문이나 if 문에서 이동 방향, 상태(IDLE, RUN, ATTACK, HURT, DEAD) 등이 결정된다

### 인벤토리
- 그리드 레이아웃 컴포넌트를 이용해 아이템이 자동으로 위치되도록 한다
- 초기화 과정에서 아이템을 표시할 영역을 미리 배치하고 실제 아이템 표시는 해당 위치에 아이콘을 띄운다
- 아이템을 배치할 때는 비어 있는 슬롯 중 가장 빠른 슬롯부터 채운다
    - 최초에 0번 슬롯부터 시작
- 드래그 앤 드랍으로 아이템을 인벤토리에서 제거할 수 있다
- 아이템을 더블 클릭하면 착용했다는 의미로 배경색이 변경되고 해제하면 다시 원래대로 돌아간다

### 상태창
- 인벤토리에서 아이템을 더블 클릭하면 해당 아이템의 타입에 맞는 동작이 수행된다
    - 무기는 손, 방어구는 몸통, 헬멧은 머리, 부츠는 발 영역에 표시된다
    - 물약은 소비되고 회복양 만큼 HP가 회복된다

### 애니메이션
- 트랜지션을 이용해 타격 애니메이션이 끝나면 타격 직전의 애니메이션으로 돌아간다
- HP가 0이 되면 사망 애니메이션이 재생되고 애니메이션이 종료되면 이벤트가 발생되어 서버에 알린다
    - 서버에서 사망 이후의 후속 로직이 실행

### Protobuf 사용
- C++ 서버와의 원활한 통신을 위해 protobuf를 사용한다
- 코드 자동화와 bat 파일 등을 사용해 패킷이 변경되면 서버와 클라이언트에 동시에 반영될 수 있도록 한다
